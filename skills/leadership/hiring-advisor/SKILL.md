## Purpose

Guide PMs through hiring for product roles — from **role definition and job descriptions to interview design, scorecard creation**, and making objective hiring decisions.

---

## Application

### Turn 1: Define the Role

**Agent asks:** "What role are you hiring for?"
- Level (APM / PM / Senior PM / Director)
- Scope (what they'll own)
- Team context (existing team, new team, backfill)
- Must-have experience vs. nice-to-have

### Turn 2: Job Description

```markdown
## Job Description: [Title]

### About the Role
[2-3 sentences: what this person will do and why it matters]

### You Will
- [Responsibility 1 — outcome-focused, not task-focused]
- [Responsibility 2]
- [Responsibility 3]
- [Responsibility 4]

### You Have
- [Required qualification 1 — be specific about experience level]
- [Required qualification 2]
- [Required qualification 3]

### Bonus
- [Nice-to-have 1]
- [Nice-to-have 2]

### Why Join Us
- [Compelling reason 1]
- [Compelling reason 2]
```

### Turn 3: Interview Design

```markdown
## Interview Loop

| Stage | Interviewer | Focus Area | Duration |
|-------|-----------|-----------|----------|
| Screen | Recruiter | Culture fit, basic quals | 30 min |
| PM Screen | Hiring PM | Product sense, motivation | 45 min |
| Product Sense | PM Peer | Product design question | 45 min |
| Execution | Eng Lead | Execution, technical collaboration | 45 min |
| Analytical | Data/Analytics | Metrics, data sense | 45 min |
| Leadership | Skip-level manager | Strategy, leadership | 45 min |
| Culture | Cross-functional | Values, collaboration | 30 min |

## Interview Scorecard
| Dimension | 1 (No) | 2 (Weak) | 3 (OK) | 4 (Strong) | 5 (Exceptional) |
|-----------|--------|---------|--------|-----------|----------------|
| Product Sense | | | | | |
| Execution | | | | | |
| Analytical | | | | | |
| Leadership | | | | | |
| Culture Fit | | | | | |

**Hire bar:** Average ≥3.5, no dimension below 3
```

### Turn 4: Decision Framework

```markdown
## Hiring Decision

### Strong Hire Signals
- [Signal 1 — e.g., "Structured their answer with a framework"]
- [Signal 2 — e.g., "Asked great clarifying questions"]
- [Signal 3 — e.g., "Showed data-driven thinking"]

### Red Flags
- [Flag 1 — e.g., "Couldn't explain a product decision they made"]
- [Flag 2 — e.g., "No curiosity about our product/users"]
- [Flag 3 — e.g., "Blamed team for failures"]
```
