## Purpose

Map your entire business model on a single page using the **9 building blocks** of Osterwalder's Business Model Canvas. Clarifies how your product creates, delivers, and captures value — essential for strategic alignment and identifying model risks.

---

## Key Concepts

### The 9 Building Blocks

```
┌──────────┬──────────┬──────────┬──────────┬──────────┐
│ KEY      │ KEY      │ VALUE    │ CUSTOMER │ CUSTOMER │
│ PARTNERS │ ACTIVI-  │ PROPOSI- │ RELATION-│ SEGMENTS │
│          │ TIES     │ TION     │ SHIPS    │          │
│ Who      │ What     │          │ How      │ Who      │
│ helps us │ we do    │ What we  │ we keep  │ we serve │
│          │          │ deliver  │ them     │          │
│          ├──────────┤          ├──────────┤          │
│          │ KEY      │          │ CHANNELS │          │
│          │ RESOURCES│          │ How we   │          │
│          │ What we  │          │ reach    │          │
│          │ need     │          │ them     │          │
├──────────┴──────────┴──────────┴──────────┴──────────┤
│ COST STRUCTURE                │ REVENUE STREAMS       │
│ What we spend                 │ How we earn           │
└───────────────────────────────┴───────────────────────┘
```

## Application

### Fill Each Block

```markdown
# Business Model Canvas: [Product Name]

## Customer Segments
- [Primary segment: who specifically?]
- [Secondary segment: who else?]
- [Who is explicitly NOT a target?]

## Value Proposition
- [Core value for each segment]
- [What job does it do for them?]
- [What pain does it relieve?]

## Channels
- [How customers discover you]
- [How they evaluate you]
- [How they buy]
- [How you deliver value]
- [Post-purchase support channels]

## Customer Relationships
- [Self-service / Assisted / Dedicated]
- [Community / Co-creation]
- [Retention mechanisms]

## Revenue Streams
- [Pricing model: subscription / usage / one-time]
- [Price points]
- [Upsell/cross-sell paths]

## Key Resources
- [Technology / IP / Data]
- [People / Expertise]
- [Brand / Network]

## Key Activities
- [Core product activities]
- [Sales/marketing activities]
- [Support activities]

## Key Partnerships
- [Strategic alliances]
- [Technology partners / Integrations]
- [Channel partners]

## Cost Structure
- [Fixed costs]
- [Variable costs]
- [Key cost drivers]
- [Cost structure type: cost-driven vs. value-driven]
```

## Examples

### ✅ Good Value Proposition Block

```markdown
## Value Proposition
- **For PM teams:** Unified product metrics without SQL — save 5+ hours/week
- **For executives:** Real-time product health dashboard for board meetings
- **Pain reliever:** Eliminates manual data export and spreadsheet compilation
- **Gain creator:** Enables data-driven decisions in meetings, not after them
```

### ❌ Bad Value Proposition Block

```markdown
## Value Proposition
- We're an analytics platform
- We have AI features
```

## Quality Rubric

| Dimension | 0 | 1 | 2 |
|-----------|---|---|---|
| **Completeness** | 3-4 blocks | 7-8 blocks | All 9 blocks filled |
| **Specificity** | Generic entries | Named but surface | Specific with evidence |
| **Coherence** | Blocks contradict | Mostly aligned | All blocks reinforce each other |
| **Revenue/Cost** | Missing | Listed but not analyzed | Unit economics considered |
| **Risks** | Unstated | Acknowledged | Key model risks identified |

---

## Common Pitfalls

### Pitfall 1: Canvas as a Static Document

**Fix:** Review and update quarterly. Business models evolve — the canvas should too.

### Pitfall 2: Value Prop Doesn't Match Segments

**Fix:** Each customer segment should map to a specific value proposition. If your VP is generic, you're talking to "everyone" (which means no one).
