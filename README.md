## 🔥 Why PM Skills?

Most PMs use AI like a search engine — they type a vague prompt and get a vague answer. **PM Skills turns any AI into a senior product management partner.**

> **Prompts are one-shot. Skills are permanent upgrades.**

| Without PM Skills | With PM Skills |
|:-:|:-:|
| "Write me a PRD" | Agent asks 7 clarifying questions, produces a 10-section PRD, self-scores quality |
| "Help me prioritize" | Agent recommends RICE vs Kano vs MoSCoW based on *your* context |
| "Build a strategy" | Agent runs a full 6-phase strategy session using Moore, Osterwalder & OKRs |
| Outputs vary wildly | Consistent, rubric-graded professional output every time |
| You're the expert, AI is the typist | **AI becomes your thinking partner** |

## 📐 Skill Format

Every skill follows a **consistent enhanced structure** with YAML frontmatter:

```yaml
---
name: skill-name
type: component | interactive | workflow
domain: discovery | strategy | planning | ...
difficulty: beginner | intermediate | advanced
estimated_time: "10-30 min"
prerequisites: [dependent-skills]
outputs: ["artifacts-produced"]
triggers: ["phrases that activate this skill"]
---
```

### Every Skill Includes

| Section | What It Does |
|---------|-------------|
| **Purpose** | When and why to use this skill |
| **Key Concepts** | Core frameworks and mental models |
| **Application** | Step-by-step with fill-in templates |
| **Quality Rubric** | 0–2 scoring across key dimensions |
| **Common Pitfalls** | Named failure modes with fixes |
| **References** | Cross-references to related skills |

## 💡 Philosophy

```
  🎯 Outcomes over outputs     — solve problems, don't ship features
  📊 Evidence over vibes       — validate with data, not opinions
  ✂️  Clarity over completeness — simple and usable beats comprehensive and confusing
  👁️ Show, don't tell          — examples beat explanations
  🚫 Anti-patterns matter      — knowing what NOT to do is half the battle
```
